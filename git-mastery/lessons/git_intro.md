1. What is Git? (DevOps-friendly intro)

Git is a version control system.

In simple words: Git helps you track changes in your code over time. 

2. What is Version Control?

Version control is a system that tracks changes to files over time.

3. Difference between Centralized vs Distributed Version Control

  - Centralized Version Control (CVCS)

    Definition:

    There is one central server that stores the entire project history.

    How it works ;

      You must connect to the server
  
      You get the latest files from the server
  
      Changes are sent back to the server

  - Distributed Version Control (DVCS)

    Definition:
    
    Every developer has a full copy of the repository and its history.

    How it works:

      You clone the repository
      
      You work locally
      
      You commit without internet
      
      Later, you sync with others
